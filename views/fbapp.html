Development

The game was originally planned to be released as a stand-alone chatbot for Facebook Messenger. However, to be able to fully test the game without having multiple facebook accounts, a web-client was implemented.

The server handles all client connections through the chat. Several structured messages have been implemented to minimize the number of commands the user needs to learn to interact with the game.

However, some commands were implemented in order for users to be able to poll the server for information they might need. All commands are preceded by a '.' to distinguish them from messages that are meant to be sent to other users.

Commands

  • .exit - Prompts the user if he wants to leave the current game. The user receives a simple structured message.
  • .help - Sends the user a text message with each one of the game commands and their purpose.
  • .role - Reminds the user what his current game role is.
  • .roles - For each role that has been assigned in the current game, if the role has already been revealed, it shows the alias of the player who holds the role.
  • .alive - Alias of all the players that are still alive in the current game.
  • .dead - Alias of each player that has been killed in the current game.
